# B2B Delivery IQ — quick briefing (all roles)

**What it is:** Wholesale retailers order from distributors; orders sit in **time-boxed batch windows**; when a batch closes, the system builds delivery **runs** (pickups → drops). **AI** helps with **natural-language ordering** and **choosing between two pre-scored route plans** — routing itself is **not** generated by the LLM.

---

## Start the stack (minimal)

1. **MySQL** running; create a database.  
2. **Backend** (`hackathon-backend/`): `cp .env.example .env` → set `DATABASE_URL`, `JWT_SECRET` (≥16 chars), `FRONTEND_URL`.  
3. `npm install` → `npm run migrate` → `npm run seed:delivery-sites` → optional `npm run seed:mock-products`.  
4. `npm run dev` → API at **http://localhost:8000** · docs at **/api/docs**.  
5. **Frontend** (`hackhathon-frontend/`): `.env.local` with `NEXT_PUBLIC_API_URL=http://localhost:8000` → `npm install` → `npm run dev` → **http://localhost:3000**.

## Roles at a glance

| Role | Register as | Main UI | Must do before “full” demo |
|------|-------------|---------|------------------------------|
| **Retailer** | Retailer | Catalog, Cart, Orders | Pick **delivery site** on Cart |
| **Distributor** | Distributor | Dashboard, stock/products | Set **depot site** on dashboard |
| **Delivery partner** | Delivery partner | Runs, Vehicle | Set **capacity (kg)** on Vehicle page |

---

## Where AI helps

| Feature | Needs API key? |
|---------|----------------|
| NL order preview / catalog chat | Yes — `OPENAI_API_KEY` on **server** |
| Pick plan A vs B after batch close | Optional — **fallback** picks best score without AI |

---

## Batch timer (demos)

Set in **backend** `.env**: `BATCH_WINDOW_MINUTES=2` (and restart). Countdown in the app uses **server** `closes_at`.
